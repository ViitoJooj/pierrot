— A linguagem .pierrot

Reatividade

O Pierrot não tem virtual DOM nem eval. O estado são só as variáveis de topo do <script>. Toda interação chama __pierrotUpdate() no final, e o runtime repinta apenas o que depende do estado.

A cada update: spans ${var} recebem o novo valor; blocos @for / @if re-rodam; inputs com @bind recebem o valor (exceto o focado); @render é repreenchido. Esta página cobre @event, @bind e @render. Interpolação e blocos estão em Sintaxe de template.

@event={fn} — eventos

Qualquer evento do DOM vira uma diretiva @<evento>:

@click={increment} vira onclick="increment(); __pierrotUpdate()". Depois que o handler roda, o update repinta ${count}. Funciona com qualquer evento: @input, @submit, @mouseover, @scroll, etc.

Com argumentos

  • @click={fn} → fn()
  • @click={fn(args)} → fn(args)

Num componente, passar uma prop como argumento é o jeito de levá-la à lógica (veja props no script).

@bind={var} — two-way binding

Liga um <input> ou <textarea> a uma variável nos dois sentidos:

  • Digitar no input atualiza name e dispara __pierrotUpdate().
  • Mudar name no código atualiza o valor do input.
  • O elemento focado não é sobrescrito enquanto você digita.

@render — HTML, Markdown e Pierrot

Renderiza conteúdo dinâmico que não é escapado como texto. Fica na sua própria linha.

@render html(expr)

Injeta HTML cru (o resultado de expr) no lugar. Reavaliado a cada update. Útil para syntax highlight, conteúdo gerado, etc.

A segurança é sua responsabilidade: HTML cru não é escapado. Não injete input de usuário não confiável.

@render markdown(expr)

Converte Markdown em HTML no browser. Suporta títulos, listas, blocos de código e inline (**bold**, *itálico*, `code`, [link](url)).

@render pierrot(expr | "literal")

Renderiza um trecho de código .pierrot — um mini-componente vivo.

  • String literal: resolvido em tempo de compilação. O <script> do trecho entra no bundle da página e o template é inlined.
  • Expressão: vira um iframe sandboxed (allow-scripts) recompilado no browser quando o valor muda (debounce de 300ms). É assim que este site faz o preview vivo do editor:

Como tudo se conecta

Digitar no input → query muda → __pierrotUpdate() → o bloco @for reavalia filtered() e re-renderiza a lista. Sem virtual DOM, sem framework de gerência de estado.

→ Helpers disponíveis no <script> (get, client, time): Script API.
