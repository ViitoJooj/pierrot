Estrutura do projeto

Onde os arquivos ficam, como pastas viram rotas e a anatomia comum a todo .pierrot — página, componente ou layout.

Layout de pastas

Os caminhos não são mágicos: o entry point é app.entry do settings.pierrot.json (default ./src/main.pierrot), e a pasta dele vira a raiz src do projeto. Todo o resto é relativo a essa pasta.

→ Veja configuração para mudar esses caminhos.

Rotas = pastas em pages/

Cada pages/<rota>/index.pierrot vira uma rota. O nome da pasta é o caminho:

PastaRota
pages/home/index.pierrotdepende de set.Default (geralmente /)
pages/about/index.pierrot/about
pages/blog/post/index.pierrot/blog/post

A rota / aponta para a página marcada com set.Default(...) no layout. Rotas inexistentes caem na página set.Fallback(...) com status 404.

Anatomia de um arquivo .pierrot

Todo .pierrot — página, componente ou layout — tem o mesmo formato: um <script> opcional seguido do template.

O que o parser faz com cada parte:

No <script>ViraDetalhe
import "x.css";stylesheetadicionado ao <head> (dev) ou ao bundle (build)
import "x.ts";chunk de scriptincluído no bundle da página
import { Nome } from "x.pierrot";componenteusável como <Nome /> no template
set.X(...)metadadotitle, icon, default, fallback…
let nome: tipo; (sem valor)propdeclara uma prop do componente
qualquer outro códigológicatransformado TS → JS e enviado ao browser

As linhas de import, set.X e declaração de prop são removidas do script que vai para o browser — são instruções para o compilador, não código de runtime.

Os três papéis de um .pierrot

  • Layout (main.pierrot): o entry. Tem <Slot /> (onde a página é injetada) e o set.X global. Pode envolver a página com componentes.
  • Página (pages/*/index.pierrot): o conteúdo de uma rota. Pode sobrescrever os metadados do layout.
  • Componente (components/*/index.pierrot): um pedaço reutilizável, com props.
