Documentação do Pierrot

Pierrot é um framework de componentes single-file compilado em Go. O CLI entende a linguagem .pierrot, compila tudo para HTML estático e entrega ao browser um runtime de poucos kilobytes — sem virtual DOM, sem eval, sem node_modules.

Esta é a documentação completa. Se você só quer ver algo rodando, comece pelo Hello World em 30s.

O que é o Pierrot

Você escreve a UI em arquivos .pierrot. Cada arquivo é um componente único que junta três coisas — um <script> opcional (imports, metadados, props e lógica em TypeScript) e o template (HTML mais diretivas):

O servidor (pierrot dev) ou o build (pierrot build) lê esse arquivo, resolve imports e componentes, transforma o TypeScript em JavaScript com esbuild e monta um documento HTML completo. O browser recebe HTML pronto mais um runtime pequeno que reavalia as partes dinâmicas quando um evento muda o estado.

Não há etapa de build de JS no seu projeto, sem package.json, sem virtual DOM. O estado são apenas as variáveis de topo do <script>; cada @event chama __pierrotUpdate() no final para repintar os spans e blocos afetados.

Arquitetura

Todo o compilador vive em internal/. É um pipeline curto de funções puras que roda por request no pierrot dev e uma vez por página no pierrot build. Sem AST gigante, sem worker pool, sem cache.

#EtapaArquivoO que faz
1parsereaders/parser.goSepara o <script> do template. Extrai imports de CSS/TS/componente, metadados set.X e props.
2expansãoworkers/dev_server.goRecursivo: layout → página → componentes. <Slot /> recebe a página; cada <Nome /> recebe o HTML do componente, props por instância.
3templateworkers/template.goBlocos @for/@if viram funções JS. @render vira placeholder. Comentários // são removidos.
4bindingsworkers/dev_server.go${var} simples vira <span data-bind>. @bind vira oninput. @event vira on<evento> + update.
5transformesbuildCada <script> e cada import "*.ts" vira um chunk TS → JS. get.Dotenv("X") é trocado pelo valor literal antes.
6montagemworkers/dev_server.goHTML final: <head> com title/meta/links de CSS, <body> com o markup, e o runtime gerado.

O runtime do browser

O runtime não usa eval. O servidor já conhece os nomes usados em ${...} e gera um objeto state a partir deles. A cada __pierrotUpdate():

  • spans data-bind recebem o valor da variável;
  • blocos data-pierrot-block (@for/@if) re-rodam sua função JS e re-renderizam;
  • spans data-pierrot-expr (expressões compostas) são reavaliados;
  • inputs com @bind recebem o valor (pulando o elemento focado);
  • placeholders @render são repreenchidos (html / markdown / iframe de preview).

Convenções rápidas

  • Rotas = pastas em src/pages/. pages/about/index.pierrot → rota /about.
  • A página default (/) e a fallback (404) são definidas no layout com set.Default(...) e set.Fallback(...).
  • Componentes começam com letra maiúscula (<Header />); tags minúsculas são HTML puro.
  • Props são declaradas no componente como let nome: tipo; sem valor.
  • Assets (imagens, fontes, robots.txt, favicon.ico) ficam em src/assets/.
