Script API

Dentro do <script> de qualquer .pierrot você ganha, além de TypeScript puro, quatro objetos do framework: set (metadados, em tempo de compilação), get, client e time (runtime no browser). Mais acesso às variáveis do .env.

set — metadados

set.X(...) configura a página. Faz sentido no layout (main.pierrot) e nas páginas — uma página pode sobrescrever o que o layout definiu. As chamadas são resolvidas em compilação e removidas do script.

ChamadaEfeito
set.Title("...")<title>
set.Description("...")<meta name="description">
set.Charset("...")<meta charset> (default UTF-8)
set.Icon("./assets/favicon.ico")<link rel="icon">
set.Robots("./assets/robots.txt")arquivo servido em /robots.txt
set.Default(Componente)página da rota /
set.Fallback(Componente)página servida em rotas inexistentes (404)

set.Default e set.Fallback recebem um componente importado, e a pasta dele (relativa a pages/) vira a rota. O import precisa apontar para dentro de pages/.

get — info do request

ChamadaRetorna
get.Path(n)o n-ésimo segmento do pathname, 1-based. /about/team → get.Path(1) == "about". Fora do range: "".
get.Status()status HTTP como string: "200", "404".
get.Dotenv("NOME")valor de uma variável do .env (veja abaixo).

client — navegação

  • client.Redirect(url) — navega para url.
  • client.Redirect(url).newTab() — abre em nova aba em vez de navegar.

time — espera assíncrona

time.Sleep(n) retorna uma Promise que resolve após n unidades. Escolha a unidade encadeando:

Unidades disponíveis: .msec(), .sec(), .min(), .hr(), .day(), .week(), .month(), .year().

O compilador injeta o await e torna a função async automaticamente quando vê time.Sleep(...). Depois que o código após o await roda, um __pierrotUpdate() dispara — então estado mudado pós-sleep já está na tela.

Como funciona, e os limites:

  • A substituição acontece no servidor, em compilação: get.Dotenv("API_URL") vira o valor literal antes do bundle. Não existe em runtime.
  • Só as variáveis referenciadas terminam no HTML — o resto do .env nunca chega ao browser.
  • O argumento precisa ser uma string literal entre aspas duplas. Um nome dinâmico é erro — a substituição é textual.
  • Uma variável ausente do .env, ou dotenv desabilitado, vira "" + um erro no overlay/build.

⚠️ Como o valor é embutido no HTML enviado ao browser, não coloque segredos (chaves de API privadas, senhas) em variáveis lidas por get.Dotenv. Trate tudo que passa por ele como público.
