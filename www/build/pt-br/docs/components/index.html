Componentes & props

Um componente é um .pierrot reutilizável. Mesma anatomia de uma página: um <script> opcional e um template. A diferença é que ele declara props e é usado como tag (<Nome />) dentro de outro .pierrot.

Usando um componente

Importe dentro do <script> de quem usa, depois solte a tag no template. Componentes começam com letra maiúscula; o nome em import { Nome } é o nome da tag.

  • A tag precisa ser self-closing: <Nome ... />. Não há tag de fechamento separada.
  • Componentes podem importar outros componentes — a expansão é recursiva.
  • Se sobrar uma tag maiúscula no HTML final (sem import), o Pierrot reporta tag <Nome /> não corresponde a nenhum componente importado.

Props

Uma prop é declarada no componente como let nome: tipo; sem valor. Quem usa o componente passa o valor como atributo da tag.

Literal vs. expressão

No usoSignificadoComo chega
title="Olá"literalinserido como texto puro no HTML do componente
price={valor}expressãovira ${valor}, avaliado no browser

No template do componente, cada referência à prop ({title}, ${price}) é trocada pelo valor da instância. Uma prop declarada mas não passada vira "".

Props em atributos

Uma prop como expressão dentro de um atributo só funciona dentro de um bloco @for / @if:

Fora de um bloco, uma expressão num atributo viraria um <span> dentro do atributo (quebrado). Use uma prop literal ali, ou mova para dentro de um bloco.

Props no <script> do componente

O <script> de um componente é compartilhado entre todas as instâncias — por isso uma prop não vira variável dentro dele. O único jeito de uma prop chegar à lógica do componente é como argumento de um @event ou @render:

Limites das props, resumidos:

1. Uma prop não vira variável no <script> (passe como argumento de event/render).
2. Uma prop expressão num atributo só funciona dentro de @for / @if.
3. Uma prop não passada = "".

Composição e CSS

  • Cada componente importa seu próprio styles.css. O CSS é coletado uma vez (sem duplicar) mesmo que o componente apareça várias vezes.
  • No pierrot dev, cada CSS vira um <link>. No pierrot build, todos viram um único bundle.css minificado por página.
