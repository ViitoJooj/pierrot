— The .pierrot language

Reactivity

Pierrot has no virtual DOM and no eval. State is just the top-level variables of the <script>. Every interaction calls __pierrotUpdate() at the end, and the runtime repaints only the bits that depend on the state.

On each update: ${var} spans get the new value; @for / @if blocks re-run; inputs with @bind get the value (except the focused one); @render is refilled. This page covers @event, @bind and @render. Interpolation and blocks are in Template syntax.

@event={fn} — events

Any DOM event becomes an @<event> directive:

@click={increment} becomes onclick="increment(); __pierrotUpdate()". After the handler runs, the update repaints ${count}. Works with any event: @input, @submit, @mouseover, @scroll, etc.

With arguments

  • @click={fn} → fn()
  • @click={fn(args)} → fn(args)

In a component, passing a prop as an argument is the way to get it into the logic (see props in the script).

@bind={var} — two-way binding

Links an <input> or <textarea> to a variable both ways:

  • Typing in the input updates name and fires __pierrotUpdate().
  • Changing name in code updates the input's value.
  • The focused element is not overwritten while you type.

@render — HTML, Markdown and Pierrot

Renders dynamic content that is not escaped as text. Goes on its own line.

@render html(expr)

Injects raw HTML (the result of expr) in place. Re-evaluated on every update. Useful for syntax highlighting, generated content, etc.

You are responsible for safety: raw HTML is not escaped. Do not inject untrusted user input.

@render markdown(expr)

Converts Markdown to HTML in the browser. Supports headings, lists, code blocks, and inline (**bold**, *italic*, `code`, [link](url)).

@render pierrot(expr | "literal")

Renders a snippet of .pierrot code — a live mini-component.

  • String literal: resolved at compile time. The snippet's <script> goes into the page bundle and the template is inlined.
  • Expression: becomes a sandboxed iframe (allow-scripts) recompiled in the browser when the value changes (300ms debounce). That is how this site does the live editor preview:

How it all connects

Typing in the input → query changes → __pierrotUpdate() → the @for block re-evaluates filtered() and re-renders the list. No virtual DOM, no state management framework.
