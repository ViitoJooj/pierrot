Components & props

A component is a reusable .pierrot. Same anatomy as a page: an optional <script> and a template. The difference is that it declares props and is used as a tag (<Name />) inside another .pierrot.

Using a component

Import it inside the <script> of whoever uses it, then drop the tag in the template. Components start with an uppercase letter; the name in import { Name } is the tag name.

  • The tag must be self-closing: <Name ... />. There is no separate closing tag.
  • Components can import other components — expansion is recursive.
  • If an uppercase tag is left over in the final HTML (no matching import), Pierrot reports tag <Name /> doesn't match any imported component.

Props

A prop is declared in the component as let name: type; with no value. Whoever uses the component passes the value as a tag attribute.

Literal vs. expression

In the usageMeaningHow it arrives
title="Hello"literalinserted as plain text into the component's HTML
price={value}expressionbecomes ${value}, evaluated in the browser

In the component's template, each reference to the prop ({title}, ${price}) is replaced by the instance's value. A prop declared but not passed becomes "".

Props in attributes

A prop as an expression inside an attribute only works inside a block @for / @if:

Outside a block, an expression in an attribute would turn into a <span> inside the attribute (broken). Use a literal prop there, or move it inside a block.

Props in the component's <script>

A component's <script> is shared across all instances — that is why a prop does not become a variable inside it. The only way a prop reaches the component's logic is as an argument to an @event or @render:

Prop limits, summarized:

1. A prop doesn't become a variable in <script> (pass it as an event/render argument).
2. An expression prop in an attribute only works inside @for / @if.
3. A prop not passed = "".

Composition and CSS

  • Each component imports its own styles.css. CSS is collected once (no duplicates) even if the component appears multiple times.
  • Under pierrot dev, each CSS becomes a <link>. Under pierrot build, they all become a single minified bundle.css per page.
