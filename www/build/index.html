Componentes em um arquivo.

Pierrot é um framework de componentes single-file. O CLI escrito em Go compila .pierrot direto para HTML; o browser recebe um runtime de poucos kilobytes, sem virtual DOM, sem eval, sem npm.

$

go install github.com/pierrot/cmd/pierrot@latest

ler a documentação → hello world em 30s

— 01 / POR QUÊ

Três coisas
que o Pierrot não tem.

Cada decisão do framework começa em remoção. O resultado é um stack que cabe na cabeça em uma tarde: um binário Go, um arquivo por componente, um runtime que cabe em poucos kilobytes.

◆◆◆

— 02 / QUICKSTART

Do zero ao
localhost em 30 segundos.

Sem template repo no GitHub, sem create-pierrot-app. Um binário, três comandos. Salvou o arquivo, o browser recarrega.

— 05 / ARQUITETURA

O que acontece
entre .pierrot e <html>.

Um pipeline curto, todo em Go. Cada etapa é um arquivo no internal/. Sem AST gigante, sem worker pool, sem cache — funções puras que rodam por request no dev e uma vez no build.
