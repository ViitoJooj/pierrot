v0.1 · front-end framework in Go

Components in one file.

Compiled in Go.

Pierrot is a single-file component framework. The Go CLI compiles .pierrot straight to HTML; the browser gets a runtime of a few kilobytes, no virtual DOM, no eval, no npm.

$

go install github.com/pierrot/cmd/pierrot@latest

read the documentation → hello world in 30s

— 01 / WHY

Three things
Pierrot does not have.

Every framework decision starts with removal. The result is a stack that fits in your head in an afternoon: one Go binary, one file per component, a runtime that fits in a few kilobytes.

◆◆◆

— 02 / QUICKSTART

From zero to
localhost in 30 seconds.

No GitHub template repo, no create-pierrot-app. One binary, three commands. Save the file, the browser reloads.

— 05 / ARCHITECTURE

What happens
between .pierrot and <html>.

A short pipeline, all in Go. Each step is a file under internal/. No giant AST, no worker pool, no cache — pure functions that run per request in dev and once at build.
